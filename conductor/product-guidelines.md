# Product Guidelines

## Tone & Voice
The system should employ a **Friendly & Conversational** tone in all user-facing interactions. While the underlying technology is complex, the interface should feel approachable and use natural language rather than overly formal or technical phrasing.

## UI/UX Principles
The dashboard and interfaces must be **Data-Dense & Compact**. Given the goal of managing and analyzing large volumes of leads and interactions, the UI should maximize the amount of information (such as contact details, scores, and data points) visible on a single screen without requiring excessive scrolling or clicking.

## Data Handling
The system should default to **Aggressive Extraction**. To build the most comprehensive networking graph possible, the AI is authorized to automatically maximize data extraction from all connected sources (Telegram, Excel, etc.) into structured profiles.

## Error Handling
When issues occur (e.g., API timeouts or failed extractions), the system should display **User-Friendly Alerts**. Avoid showing verbose technical jargon or stack traces to the end user; instead, provide simple, actionable messages that explain what went wrong and how they can proceed.
